# Team video (≤ 3 minutes) — outline

The submission form asks for a ≤3 minute team video covering: team, course, university, dev strategy, how you used IBM Granite, and IBM SkillsBuild badges.

## Suggested structure (2:30–3:00)

0:00–0:15 — Title + who you are
- University + course
- Team name
- Team members + roles (1 sentence each)

0:15–0:55 — Problem + goal
- “We built an autonomous driver for TORCS (Corkscrew, standing start, 1 lap).”
- Show 1–2 seconds of the car driving.

0:55–1:35 — Technical approach (show code + config)
- Rule-based controller in `src/driver_baseline.py`
- Segment map YAML (`telemetry/segments_baseline_current.yaml`)
- Explain lookahead braking + racing line at a high level

1:35–2:15 — IBM Granite usage (show Continue/Ollama)
- Granite used for code review + telemetry interpretation (offline)
- Show VS Code + Continue with Granite model selected
- One concrete example: “We pasted a segment report and Granite suggested a safer apex/exit line change.”

2:15–2:40 — SkillsBuild badges
- Show 1–2 badge screenshots
- 1 sentence each: what you learned + how it helped the project

2:40–3:00 — Close + links
- Final lap time highlight (standing start)
- “Repo + blog links are in the submission form.”

## Recording tips
- Good lighting and clean audio (quiet room)
- Keep cuts tight; show the code/config on screen while narrating
- Ensure the video is shared as “anyone with link can view” (Google Drive).
